Project layout https://github.com/golang-standards/project-layout

Pastas importantes:

internal -> códigos específicos do projeto e do negócio, não públicas
pkg -> pasta que é o oposto à pasta internal
cmd -> pasta que contém o executrável do projeto
configs -> configurações ou helpers de configurações
test -> arquivos adicionais que auxiliam nos testes, não os testes em si
api -> documentações e especificações de api
